# DEMO-WORKFLOW — 시청자 관점 데모 제작 워크플로우 (사용자 피드백 반영, 2026-07-07)

원칙: **판정 기준은 게이트가 아니라 "사람이 봤을 때"다.** 게이트는 최소선일 뿐, 최종 판정은 시청자 경험 QC.

## 시청자 경험 기준 (전 데모 공통, 위반 = 반려)

1. **정지 금지 — 동적 최우선**: 어느 1.5초 구간에도 화면 전체가 멈춰 있으면 안 됨. 씬 타임라인은 3단 구성 의무 — 엔트런스(0~25%) → **리빙 모션**(중반: 펄스·드리프트·패럴랙스·카운트 지속·배경 흐름) → 엑싯/핸드오프(말미 15%). 블록 엔트런스 후 홀드는 금지.
2. **배경도 산다**: 정적 그라데이션 금지 — 드리프트하는 그라데이션·글로우 스윕·미세 파티클 등 상시 배경 모션. 이미지 씬은 켄번즈 상시.
3. **카피 가독**: 헤드라인 1.5초 내 완독(12자 이내), 대비는 시청 거리 기준(작은 라벨도 화면에서 읽혀야) — 계층색(제목/항목/라벨) 뚜렷하게.
4. **이미지 실사용**: 생성 이미지가 씬에 실제로 보여야 함(배경·풀블리드·사이드) — 이미지 0장 데모 금지.
5. **선명도**: 차트·텍스트 뭉개짐 금지 — 스케일 변환으로 인한 블러 검출 시 원인 제거(네이티브 해상도 렌더).
6. **페이싱**: 컷 2~4초, 전환은 임팩트 있게(플래시·와이프 속도감), 30초 안에 지루한 구간 0.

## 제작 루프

```
카피(gn-voice 윤문) ─┐
이미지(imagegen)   ─┼→ specs 반영 → 컴파일 → 렌더 → ① 기계 QC(빈화면·모션 밀도: 프레임간 diff로 1.5초 정지 검출)
모션 설계(3단 의무) ─┘                              → ② 시청자 QC(프레임 그리드 육안 + opus 판정: "스타 누르겠는가")
                                                   → 반려 시 원인 계층으로 루프백 → 통과분만 D 이동
```

## 기계 QC 확장 (tests/demo-visual-qc.mjs에 추가)
- 모션 밀도: 1.5초 간격 연속 프레임쌍 diff — 전 구간에서 변화율 하한 통과
- 가독: 헤드라인 bbox 대비 측정
- 이미지 사용: image-manifest의 자산이 렌더 프레임에 실출현(픽셀 매칭)
