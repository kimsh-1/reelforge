## 이 레포에서 배우지 말 것

1. Claude/Anthropic 결합을 코어 실행부에 직접 박아 넣는 방식은 피하셔야 합니다. `_coherence_fix`와 agent 실행 경로가 `anthropic` 및 Claude CLI 전제에 묶여 있어 provider 교체가 어렵습니다. 근거: `auto_agent/dashboard/scene_editor.py:1432-1481`, `auto_agent/orchestrator/runner.py:3817-3931`.

2. flat schema를 강제하면서 내부 다른 경로가 legacy 중첩 필드를 다시 쓰는 모순은 피하셔야 합니다. 한쪽에서는 `visualization.creative`를 차단하지만, 병합·기본값 경로에서는 같은 구조를 다룹니다. 근거: `auto_agent/orchestrator/runner.py:552-557`, `auto_agent/orchestrator/runner.py:2132-2145`, `auto_agent/orchestrator/runner.py:3046-3106`.

3. 공유 `manifest.json`을 편집 API에서 덮어쓰는 패턴은 프로젝트 간 오염 위험이 큽니다. 렌더 경로는 프로젝트별 props로 보완하지만, 대시보드 경로는 여전히 공유 파일을 갱신합니다. 근거: `auto_agent/dashboard/scene_editor.py:93-99`, `auto_agent/dashboard/scene_editor.py:1404-1417`, `auto_agent/orchestrator/runner.py:4419-4428`.
