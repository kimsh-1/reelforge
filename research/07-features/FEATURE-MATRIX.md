### 이 도메인 설계 조언
씬 매니페스트를 유일한 진실원으로 두고, storyboard/index/cache/selection snapshot은 provenance가 붙은 파생 산출물로만 취급하라.  
duration/fps/seed/media range는 렌더 전 물리 하드게이트로 고정하고, 부분 재렌더 invalidation은 sceneNumber가 아니라 stable sceneId와 asset provenance로 계산하라.  
한국어 TTS의 word/char/speaker timestamp는 후처리 자막 파일이 아니라 scene timeline의 1급 필드로 두어 자막, 씬 분할, HTML 렌더가 같은 시간을 보게 하라.

### 이 도메인 설계 조언

자막은 text가 아니라 `words[] + cues[] + style + source + validationReport`를 가진 scene manifest track으로 고정해야 부분 재렌더가 흔들리지 않는다.  
한국어는 provider 워드 이벤트를 그대로 쓰지 말고 조사·어절·문장부호 분절과 실제 폭 기반 wrap을 컴파일러의 결정론 규칙으로 둬야 한다.  
렌더 전 폰트, cue duration, overflow, 마지막 프레임, SRT parse, transcript drift를 물리 하드게이트로 막고 Studio는 같은 리포트를 보여줘야 한다.

### 이 도메인 설계 조언
1. `utterance -> audio_request -> audio_meta -> cue`를 하나의 canonical 계약으로 묶고, provider별 차이는 어댑터 내부에만 둬라.
2. 최종 렌더의 시간축은 반드시 실측 오디오 메타에서 오게 하고, 텍스트 길이 추정은 preview 전용으로만 허용해라.
3. 한국어 TTS는 전처리 diff, timestamp capture, loudness/sync/path 하드게이트를 한 파이프라인으로 묶어야 부분 재렌더가 안정된다.

### 이 도메인 설계 조언
P0는 “렌더 전 차단 가능한 결정론 검증”으로 좁혀야 한다. LLM judge, 대시보드 컴포넌트, 디버그 API는 보조층이다.

게이트 결과는 모두 동일한 JSON envelope로 통합하라: `gateId`, `phase`, `severity`, `blocking`, `evidence`, `repairHint`.

한국어 TTS/자막은 별도 품질축으로 두지 말고 text-fit, font-load, WPM, subtitle timing을 하나의 물리 하드게이트로 묶는 편이 안전하다.

### 이 도메인 설계 조언
LLM은 최종 HTML을 직접 쓰지 말고, JSON 씬 매니페스트 계약만 생성하게 한 뒤 compiler가 HyperFrames HTML로 결정론 변환해야 한다.  
프롬프트는 자유문자열이 아니라 versioned Prompt IR/YAML, source block, style block, engine compressor의 순서로 관리하는 편이 안정적이다.  
JSON repair는 허용하되 최종 권위는 schema validator와 물리 하드게이트에 두고, 스튜디오 편집은 manifest patch와 dirty draft 보호를 기본값으로 둬야 한다.
