## 이 레포에서 배우지 말 것

1. 전역 `window`에 core/engine을 노출하는 패턴은 피해야 한다. 테스트 격리, 멀티 프로젝트, 결정론 렌더 재현성을 깨뜨리기 쉽다. 근거: `src/lib/project.ts:23-26`

2. 씬/클립 계약에 `any`를 남기는 방식은 품질 하드게이트와 한국어 우선 검증에 부적합하다. manifest는 JSON Schema/Zod 등으로 닫힌 계약이어야 한다. 근거: `src/types/project.ts:35`, `src/types/timeline.ts:40-42`, `src/utils/schema-converter.ts:247-249`

3. 에디터 mount 시 프로젝트를 강제 reset하고 데모 데이터를 지연 import하는 방식은 실제 작업 상태를 덮어쓸 위험이 있다. 자동생성 대시보드는 명시적 새 프로젝트/샘플 로드 액션으로 분리해야 한다. 근거: `src/components/editor/editor.tsx:32-39`
