## 1. 한국 예능 자막 문법 — 상용 문법의 정점

한국 예능 자막은 "정보 전달"이 아니라 **감정 내비게이션 장치**다. 모든 대사·리액션·효과음을 시각적 하이라이트로 변환해 개그 포인트와 감정 비트를 증폭한다. 무한도전 분석 기준 **분당 25.5개 자막, 평균 노출 2.35초** — 이 밀도가 문법의 전제다. 이모티콘/그림 기호 사용률은 42%에 달한다.

### 1.1 자막 4종 분류 (역할 문법)

| 종류 | 역할 | 시각 처리 | ReelForge 대응 |
|---|---|---|---|
| **대사 자막 (기본형)** | 출연자 발언 그대로 | 가독성 최우선, 단일 폰트·화면 하단 고정 | 기본 subtitle 트랙 |
| **대사 자막 (강조형)** | 웃긴/중요한 발언 | 크기·색·폰트 변형 + 모션, 키워드만 색 반전 | keywordColor + scale pop |
| **상황 자막** | 장면 전환·맥락 설명 | 화면 상단/중앙, 별도 폰트(명조·붓글씨 계열 허용) | 헤드라인/카드 레이어 |
| **보조 자막** | 행동·감정 포인트 촌평("현실부정", "급정색") | 짧고 직관적, 그래픽·밈 동반, 인물 옆에 부착 | 콜아웃/스티커 레이어 |
| **설명 자막** | 구체 정보(가격·규칙·수치) | 위계 명확한 패널형, 화면 넓게 사용 | 인포 패널 컴포넌트 |

(무한도전 이후 "제3자 시점 촌평 자막"이 업계 표준이 됨 — 보조 자막이 한국 예능 문법의 고유 발명품.)

### 1.2 색 문법 (감정 → 색 매핑)

무한도전의 확립된 규칙(2008~)이 사실상 업계 원형:

| 감정/기능 | 색·처리 | 근거 |
|---|---|---|
| 상황 설명 | 검정 테두리 + 주황/하늘/검정 글자 (윤체) | 무한도전 기본 문법 |
| 대사 | 하늘색 배경 + 파랑/빨강 글자 (윤고딕 200) | 〃 |
| 미션 제시 | 검정/빨강/파랑 테두리 + 은색 글자 (수평선B체) | 〃 |
| 외침/격앙 | **노란 배경** + 구름체 | 〃 |
| 표정/감정 묘사 | **초록 테두리** 구름 배경 | 〃 |
| 쾌활·에너지 | 노랑·주황·빨강 등 난색 | 색채 연구(ECEI 2019) |
| 냉담·정색·스톱모션 대비 | 파랑 등 한색 | 〃 |
| 자연·야생·희망 | 초록 | 〃 |
| 진지·압박·미스터리 | 검정 (+흑백 화면) | 〃 |
| 억압·강한 감정 | 흑백 대비 | 〃 |

핵심 원칙: **색은 감정의 신호등**이다. 한 감정 = 한 색 조합을 프로그램 내내 고정해 시청자가 학습하게 한다(시즌 특집 때 색만 교체, 폰트는 유지 — 무한도전 2014 이후 방식).

### 1.3 테두리·배경 문법

- 한국 예능 자막은 **테두리(외곽선)가 기본값** — 어떤 배경 위에서도 읽혀야 하므로 명도 대비를 stroke로 확보. 흰 글자+검정 테두리 또는 검정 글자+흰 테두리가 뼈대.
- 강조 단계: 테두리 1겹 → 2겹(이중 스트로크) → 배경 박스 → 배경 박스+그림자+기울임. 감정 강도와 겹 수가 비례.
- 배경 박스는 "외침(노랑)", "대사(하늘)"처럼 색 자체가 의미를 갖는다. 단순 가독성 장치가 아님.

### 1.4 타이밍·기법 문법

| 항목 | 값/규칙 |
|---|---|
| 밀도 | 분당 ~25개 (예능 최상급 기준) |
| 노출 | 평균 2.35초, 리액션 촌평은 1초 내외도 허용 |
| 등장 | 발화 타이밍과 동기(박자 싱크), 강조형은 효과음과 동시 pop |
| 기법 8종 (정글의법칙 분석) | ① 광고 패러디형 ② 화면 분할 대비형 ③ 포토 프레임형 ④ 플롯 애니메이션형 ⑤ 특수효과 설명형(기류·불꽃 기호) ⑥ 등급/수치 시각화형 ⑦ 스토리텔링(책·자료 읽기)형 ⑧ 풀스크린 CG형 |
| 문장부호 | 물음표+느낌표(?!) 허용, 말줄임표는 감정 여백, 괄호는 속마음/촌평 |

## 2. 한국 유튜브 상위 채널 자막·판넬 처리

### 2.1 정보형 (슈카월드·침착맨 계열)

- **슈카월드**: 화면 = 발표 판넬(흰 배경에 이미지·차트 복붙) + 인물. 자막은 절제, 정보는 판넬이 운반. "화면 공유가 곧 그래픽"인 라이브 편집형. → ReelForge 시사점: 정보형은 자막보다 **인포 패널 레이어**가 주역.
- **침착맨**: 자막은 제공하되 BGM·효과음을 거의 안 넣는 **미니멀 편집**. 방송국식 과잉 자막의 안티테제로 성립한 스타일. 대사 자막 위주, 장식 최소. → "말 자막만 깔끔하게" 프리셋 수요의 근거.
- 공통: 정보형/교육형은 클래식 캡션(단순·일관·모든 기기에서 가독)이 정석. 폰트·색·배경·위치의 **일관성**이 채널 브랜딩.

### 2.2 방송국식 디지털 예능 (워크맨·뜬뜬·브랜드 채널)

- **워크맨**: 괄호 자막(속마음) + **모션 트래킹 자막**(인물을 따라다니는 텍스트)이 시그니처. TV 예능 문법을 유튜브 템포로 압축.
- **뜬뜬(DdeunDdeun)**: 기본형 폰트 하나를 크기·색·모션으로 변형해 감정 표현 — 폰트 수를 늘리지 않고 변형으로 다양성 확보.
- **브랜드 채널(이디야·무신사·올영TV)**: 브랜드 메인 컬러를 자막 키컬러로 고정, 채널 아트를 자막 꾸밈 요소로 재사용.
- 업계 합의 원칙(고구마팜): **명시성·가독성 > 심미성**. 폰트가 늘어나면 최소한의 브랜드 컬러 또는 유사 그래픽 스타일로 통일감을 회수.

### 2.3 폰트 실무 관행

| 용도 | 관행 폰트 (무료 상업용) |
|---|---|
| 말 자막(가독) | Pretendard, 에스코어드림, 노토산스 KR |
| 강조/외침 | 검은고딕, 배민 도현·한나·주아, Gmarket Sans Bold |
| 썸네일/헤드라인 | 여기어때 잘난체, 검은고딕, 티몬 몬소리 |
| 속도감·재치 | Rix치타 등 Rix 계열 (유료) |

규칙: 말 자막은 심플·가독, 포인트 자막은 굵고 개성 — **2폰트 체계(본문 1 + 디스플레이 1)**가 최소 안전선.

## 3. 한국 뉴스·시사 그래픽 (선거방송 = 세계적 모션 사례)

### 3.1 SBS 개표방송 ("국민의 선택")

- **바이폰(2012~)**: 득표 통계 자동 표출 영상에 패러디·유머를 입힌 템플릿 시스템. "데이터 바인딩된 모션 템플릿"의 방송판 원조 — 후보 득표율이 실시간으로 게임·영화 패러디 장면을 구동.
- **초격차 선거방송(2022)**: 초대형 미디어 월 + 고밀도 모션그래픽, 2049 시청층 압승.
- **2024**: 오징어 게임 차용 등 대중문화 패러디 지속. AI·XR까지 동원(2025 지방선거).
- 해외 파급: 후지TV가 일본 중의원 선거(2026)에 한국식 개표방송을 도입. 매 선거마다 모션그래픽 하청이 급증하는 산업 이벤트.
- **문법 요약**: ① 실데이터 → 모션 템플릿 자동 구동 ② 유머·패러디로 체류시간 확보 ③ 후보별 고정 컬러(정당색) ④ 숫자 카운트업 + 게이지 + 순위 바 차트가 기본 어휘.

### 3.2 JTBC 대비축

- JTBC는 화려함 대신 **절제된 단일 그래픽 체계 + 앵커 중심**(2017 우리의 선택). 한국 방송 그래픽의 양극: SBS(맥시멀 패러디) ↔ JTBC(미니멀 신뢰). 프리셋 설계 시 이 양극이 곧 k-variety ↔ k-broadcast 축과 동형.

### 3.3 뉴스 하단 자막·속보 관행

- **속보**: 빨간 배경 + 흰 글자, 타이틀 영역은 흰 바탕 + 빨강 "속보". 그라데이션 없는 플랫 레드가 긴급성 신호.
- **뉴스특보/NEXT**: 적색 그라데이션 밴드.
- 하단 자막바: 정보 위계(방송사 로고 → 헤드라인 → 티커) 3단 구조, 헤드라인은 흰/노랑 굵은 글자.
- 참고: 넷플릭스가 언어별 색상 자막을 도입하며 **한국어=빨강** 계열을 배정(2025) — 한국 시청자에게 색-자막 연상이 이미 학습돼 있다는 방증.

## 4. 한글 타이포그래피 모션 특성 (영문과 다른 지점)

### 4.1 구조적 차이

- 한글은 **자모 조합형 네모꼴 음절 블록** — 라틴처럼 낱글자가 흐르지 않고 음절 단위로 시각 질량이 균일. 직선·곡선 활용이 라틴보다 규칙적이라 그래픽 발산 요소가 많음.
- 획 밀도가 높아(받침 있는 음절) 같은 포인트 크기에서 라틴보다 **검게** 보인다. Black/ExtraBold에서 속공간이 뭉개지기 쉬움.
- 세로쓰기 전통 — 세로 스태거·세로 헤드라인이 자연스러운 옵션(라틴에선 부자연).

### 4.2 모션 전략 (자모 구조의 귀결)

| 전략 | 규칙 | 이유 |
|---|---|---|
| **스태거 단위** | 글자(letter)가 아니라 **음절 블록** 단위로 stagger. 어절(띄어쓰기 단위) 스태거가 차상위 | 음절이 시각 최소 단위. 자모 낱개 분해는 "타이포 아트" 모드에서만 |
| **자모 분해 연출** | ㄱ+ㅏ+ㅇ이 조립되는 어셈블리 모션은 한글 고유의 킬러 패턴 (라틴 불가능) | 조합형 문자 |
| **자간** | 제목 −2~−5%(-0.02em~-0.05em), 자막 0~−2%. 라틴식 큰 음수 트래킹 금지 | 네모꼴이라 과한 음수 자간 시 블록 충돌 |
| **행간** | 자막 1.15~1.3, 본문 1.5+, 헤드라인 1.1~1.2 | 한글은 라틴보다 x-height 개념이 없고 전각이라 행간 여유 필요 |
| **굵기** | 자막 700~800이 표준(라틴 자막의 600 상당). 900(Black)은 테두리를 얇게 하거나 생략 | 획 밀도 보상. stroke가 두꺼우면 속공간 침식 |
| **줄당 글자수** | 16자/줄, 최대 2줄 (라틴·공백·문장부호는 0.5자 계산) — Netflix 한국어 규정 | 전각 문자 폭 |
| **읽기 속도** | 성인 12자/초, 아동 9자/초 (SDH 14/11) | 〃 |
| **문장부호** | 줄 끝 마침표·쉼표 금지, 말줄임표는 U+2026 단일 문자, ?! 허용 | 〃 |
| **장평** | 모션 중 가로 스케일 90~110% 변형은 허용 범위(한글 관행) | 네모꼴이라 장평 내성이 라틴보다 큼 |
| **탈네모틀** | 샘이깊은물체류 탈네모틀 폰트는 "삐뚤빼뚤 리듬"이 필요할 때만 — 감성/수공예 톤 | 기본은 네모꼴 |

### 4.3 한글 키네틱 타이포 패턴 (ReelForge 시퀀스 후보)

1. **음절 팝(syllable-pop)** — 음절 블록별 scale 0→1.1→1 스태거(40~60ms 간격). 예능 강조 자막의 원형.
2. **자모 어셈블리(jamo-assemble)** — 초성·중성·종성이 각기 날아와 음절로 조립. 타이틀·인트로 전용, 한글 고유.
3. **받침 드롭(batchim-drop)** — 받침만 위에서 낙하해 착지+바운스. 단어 하나 강조에 효과적.
4. **어절 슬라이드(word-slide)** — 어절 단위 좌→우 슬라이드 인(읽기 방향 순응) + 마스크 와이프.
5. **외침 쉐이크(shout-shake)** — 노란 배경 박스 등장과 동시에 전체 자막 rotate ±2° 진동 3~4프레임. 예능 "외침" 문법.
6. **정색 프리즈(deadpan-freeze)** — 화면 채도 하락 + 한색(파랑) 자막이 모션 없이 하드컷 등장. 모션의 부재가 곧 개그.
7. **카운트업 게이지(broadcast-count)** — tabular 숫자 카운트업 + 득표 게이지 바. 선거방송 문법의 핵심 어휘.
8. **속보 밴드(breaking-band)** — 플랫 레드 밴드가 하단에서 슬라이드 업, 흰 굵은 글자 타이핑/컷인.
9. **트래킹 콜아웃(tracking-callout)** — 대상 좌표를 따라다니는 괄호 자막(워크맨 문법). position 키프레임 바인딩.
10. **세로 낙하(vertical-cascade)** — 세로쓰기 헤드라인이 위→아래 음절 스태거로 낙하. 라틴에 없는 옵션.

## 5. 스타일 토큰 — subtitle 16필드 번역

### 5.1 감정 상태별 토큰 (예능 문법 → 필드 오버라이드)

기본 프리셋 위에 감정 상태가 오버라이드하는 구조를 권장 (`state: shout | deadpan | mission | narration`).

| 필드 | 기본(대사) | shout(외침) | deadpan(정색) | mission(미션/공지) |
|---|---|---|---|---|
| color | #FFFFFF | #111111 | #DBEAFE | #E5E7EB(은색) |
| strokeColor | #111111 | #FACC15 없음(박스가 대비) | #1E3A8A | #DC2626 |
| strokeWidth | 3 | 0~2 | 3 | 4 (이중 느낌) |
| keywordColor | #FFD400 | #DC2626 | #60A5FA | #FACC15 |
| backgroundColor | transparent | #FFD400 | rgba(15,23,42,.85) | rgba(0,0,0,.75) |
| fontWeight | 800 | 900 | 700 | 800 |

### 5.2 프리셋 후보 1 — `k-variety` (한국 예능체)

TV 예능 + 워크맨류 디지털 예능의 공용 문법. 흰 글자 + 검정 테두리 + 노랑 키워드, 굵고 크게, 음절 팝 모션과 페어링.

```json
{
  "name": "k-variety",
  "fontFamily": "Pretendard, 'Black Han Sans', sans-serif",
  "fontSize": 52,
  "fontWeight": 800,
  "color": "#FFFFFF",
  "strokeColor": "#111111",
  "strokeWidth": 4,
  "keywordColor": "#FFD400",
  "keywordStrokeColor": "#111111",
  "backgroundColor": "transparent",
  "borderRadius": 0,
  "boxShadow": "0 4px 0 rgba(0,0,0,0.35)",
  "bottomOffset": 110,
  "maxWidth": 900,
  "lineHeight": 1.22,
  "maxCharsPerLine": 16,
  "visible": true
}
```

- 페어 모션: syllable-pop(등장), shout-shake(강조), deadpan-freeze(반전).
- 감정 오버라이드 4종(§5.1) 필수 동봉 — 색이 곧 감정 문법이므로 프리셋 단독으로는 반쪽.
- 강조 폰트 슬롯(디스플레이용): 검은고딕/Black Han Sans, 헤드라인 자간 −3%.

### 5.3 프리셋 후보 2 — `k-broadcast` (한국 방송 보도체)

뉴스·시사·선거방송 + 정보형 유튜브(슈카월드류)의 공용 문법. 배경 밴드/박스 + 테두리 없는 깨끗한 글자, tabular 숫자, 카운트업 페어링.

```json
{
  "name": "k-broadcast",
  "fontFamily": "Pretendard, 'Noto Sans KR', sans-serif",
  "fontSize": 44,
  "fontWeight": 700,
  "color": "#FFFFFF",
  "strokeColor": "transparent",
  "strokeWidth": 0,
  "keywordColor": "#FFD400",
  "keywordStrokeColor": "transparent",
  "backgroundColor": "rgba(10,21,48,0.92)",
  "borderRadius": 4,
  "boxShadow": "0 2px 12px rgba(0,0,0,0.4)",
  "bottomOffset": 88,
  "maxWidth": 1000,
  "lineHeight": 1.3,
  "maxCharsPerLine": 18,
  "visible": true
}
```

- 변형: `k-broadcast--breaking` = backgroundColor `#D71920`(플랫 레드), keywordColor `#FFFFFF`, 등장 모션 breaking-band.
- 페어 모션: broadcast-count(수치), word-slide(헤드라인), breaking-band(속보).
- 정당/카테고리 컬러 슬롯: 데이터 시리즈별 고정색 배정(선거방송 문법) — 차트 레이어와 keywordColor 동기화.

### 5.4 두 프리셋의 축

| 축 | k-variety | k-broadcast |
|---|---|---|
| 대비 확보 | stroke(테두리) | 배경 밴드/박스 |
| 감정 표현 | 색·폰트·모션 폭발 | 절제, 색은 데이터에만 |
| 밀도 | 높음(분당 20+) | 낮음(씬당 1~2) |
| 모션 성격 | 탄성·바운스·쉐이크 | 슬라이드·카운트업·와이프 |
| 원형 | 무한도전→워크맨 | SBS 국민의선택 ↔ JTBC 절제축 |

## 6. 미해결/후속

- 무한도전 원전 폰트(윤체·윤고딕200·수평선B·구름체)는 유료 — 무료 대체 매핑 검증 필요(Pretendard/검은고딕/배민 계열로 근사).
- 모션 트래킹 자막(워크맨 문법)은 좌표 바인딩 기능 필요 — 파이프라인 지원 여부 확인.
- 자모 어셈블리 패턴은 폰트 분해가 필요(오픈타입 자모 접근 or SVG 아웃라인 분해) — 기술 스파이크 대상.
