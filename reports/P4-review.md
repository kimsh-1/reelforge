# P4-R Studio Adversarial Review

Scope: `src/studio/**` and `tests/studio-e2e.test.mjs`. Method: inline Node probes using `/tmp` copies of `fixtures/golden-specs/minimal-3scene`, `startStudioServer({port:0})`, and headless Chromium through `puppeteer-core`. The repo was not used as the mutable project. Existing `tests/studio-e2e.test.mjs` was not rerun directly because it writes screenshots under `reports/studio-e2e`; the same server/panel flows were exercised without writing repo artifacts.

| 심각도 | 결함 | 실증 | 수정안 |
|---|---|---|---|
| 치명 | `/build/` 정적 서빙이 symlink를 따라 project/build 밖 파일을 노출한다. `resolveStaticPath()`는 문자열 경로가 root 안인지 확인하지만 `realpath` 검증 없이 `statSync`/`createReadStream`으로 symlink를 따라간다 (`src/studio/server/index.mjs:104`, `src/studio/server/index.mjs:125`, `src/studio/server/index.mjs:136`). | `/tmp` project의 `build/p4-symlink-secret.txt -> /tmp/.../outside-secret.txt` symlink 생성 후 `GET /build/p4-symlink-secret.txt`가 `200`과 `P4_SYMLINK_SECRET`을 반환. | `serveFile`에서 최종 파일 `realpathSync`를 계산해 served root의 realpath 하위인지 재검증하고, symlink 거부 정책이면 `lstatSync().isSymbolicLink()`도 차단한다. |
| 중대 | 기본 preview가 실제 씬을 보여주지 않는다. panel은 `/build/index.html`을 기본 source로 쓰고 (`src/studio/panel/preview.js:67`, `src/studio/panel/app.js:327`), 이 fixture의 full composition slot은 `data-composition-src`만 있고 자식 DOM이 비어 있어 중간 seek도 백지다. 동시에 scrub duration은 root `10.466s`가 아니라 GSAP transition timeline `3.167s`를 표시한다 (`src/studio/panel/preview.js:70`). | Chromium에서 iframe `/build/index.html`: `bodyTextLength=0`, slot `childElementCount=0`; `preview.seek(0.5)` 후에도 `bodyTextLength=0`. 직접 `/build/scenes/scene-s01.html`로 바꿔 `seek(0.5)`하면 `h1=오늘의 핵심`, `panelOpacity=0.8863`으로 보임. | full composition preview용 runtime이 nested scenes를 hydrate하게 하거나, 선택 씬 preview는 manifest의 scene HTML로 source를 바꾼다. duration은 `#root[data-duration]` 또는 render manifest 기준으로 표시한다. |
| 중대 | 씬 전환 시 preview source/seek가 reset되지 않는다. `selectScene()`은 state와 detail pane만 다시 그리며 preview를 갱신하지 않는다 (`src/studio/panel/app.js:420`). | `preview.seek(0.75)` 후 s02 카드를 클릭: before `{scrub:"750", selected:"s01"}`, after `{scrub:"750", selected:"s02", iframePath:"/build/index.html"}`. | `selectScene(sceneId)`에서 scene manifest path로 source를 갱신하고, 기본 seek를 entrance 이후의 safe offset 또는 0으로 명시 reset한다. |
| 중대 | editLock이 브라우저 탭 단위 잠금으로 동작하지 않는다. 서버 process owner `STUDIO_OWNER`만 사용해 같은 서버의 모든 탭이 같은 lock owner가 된다 (`src/studio/server/index.mjs:278`). | 두 Chromium 탭에서 동시에 `PATCH /api/scenes/s01` headline을 각각 A/B로 저장: 응답 status `[200,200]`, class `[E1,E1]`, 두 compile job 모두 `succeeded`, 최종 headline은 `P4 tab B headline`; lock owner는 `studio-server:<pid>`. | client/session edit token 또는 optimistic revision을 PATCH 조건으로 요구한다. 같은 process owner 재진입은 서버 내부 작업에만 허용하고 사용자 탭 간에는 409 또는 merge UI를 낸다. |
| 중대 | compile job 중복 트리거를 coalesce하지 않는다. `/api/compile`은 매 호출마다 새 job을 만들고 즉시 실행 예약한다 (`src/studio/server/index.mjs:328`, `src/studio/server/index.mjs:516`). | 연속 `POST /api/compile {scope:"full"}` 2회: HTTP `[202,202]`, job ids distinct, initial statuses `[queued,queued]`, final statuses `[succeeded,succeeded]`. | 동일 project/scope/sceneId의 running/queued compile을 재사용하거나, 이전 compile cancel/debounce 정책을 둔다. build write 구간에는 per-project mutex를 둔다. |
| 중대 | SSE 재연결은 Last-Event-ID replay가 없어 이벤트 유실이 발생한다. hub는 client set과 증가 id만 갖고 이벤트 backlog가 없으며 `connect()`가 `Last-Event-ID`를 읽지 않는다 (`src/studio/server/events.mjs:1`, `src/studio/server/events.mjs:8`). | clean fixture에서 `compile.completed id=1` 수신 후 연결 종료, `Last-Event-ID: 1`로 재연결해 700ms 수집: `replayCount=0`. | 최근 N개 이벤트 ring buffer와 Last-Event-ID replay를 구현하거나, 재연결 시 `/api/project` snapshot + job polling으로 상태 복구를 강제한다. |
| 중대 | E2/TTS 후 preview와 build duration 정합성이 깨지는 창이 있다. `/api/pipeline/tts`는 `audio_meta`를 먼저 쓰고 `tts.completed`를 emit한 뒤 compile job을 queued 상태로 반환한다 (`src/studio/server/index.mjs:533`). panel은 `tts.completed`도 preview reload 대상으로 삼아 stale build를 다시 로드할 수 있다 (`src/studio/panel/app.js:463`, `src/studio/panel/app.js:494`). | 긴 E2 입력: patch class `E2`, TTS 후 `audioDurationAfterTts=12.048`인데 `manifestDurationImmediatelyAfterTtsResponse=3.166`; compile failed 후 manifest도 `3.166` 유지. 짧은 성공 케이스도 API 응답의 compile status는 `queued`였다. | E2에서는 `tts.completed`로 preview를 reload하지 말고 `compile.completed`까지 preview/scrub을 pending 상태로 둔다. 실패 시 stale 표시와 retry CTA를 명시한다. |
| 중대 | Versions tab은 실제 gen 목록은 보여주지만 전환 버튼이 항상 404다. panel API는 `/api/versions/:resourceType` PATCH를 호출하지만 (`src/studio/panel/api.js:69`), server route에는 해당 endpoint가 없다 (`src/studio/server/index.mjs:562`). | versions UI에 `scene_specs` gens `[gen_01, gen_02, gen_03]`가 표시되고 disk dirs도 동일했지만, `전환` 클릭은 `PATCH /api/versions/scene_specs -> 404`, panel impact reason `not found`. | server에 `selectGeneration` 기반 endpoint를 추가하고, 선택 후 파일 restore/dirty 상태/preview reload까지 통합 테스트한다. |
| 낮음 | E2E가 transitions 편집 루프(E3)를 커버하지 않는다. `tests/studio-e2e.test.mjs`는 scene E1/E2, versions 표시, render만 검증하고 transitions tab 저장 경로가 없다 (`tests/studio-e2e.test.mjs:119`, `tests/studio-e2e.test.mjs:134`, `tests/studio-e2e.test.mjs:158`). | 실제 Chromium에서 transitions tab 첫 row를 `type=wipe`, `duration=0.31`로 저장: `PATCH /api/transitions` status `200`, class `E3`, actions `["compile:full"]`, compile job `succeeded`, disk transition 반영. | E2E에 transitions tab edit/save/assert를 추가하고, compile 완료 및 manifest transition type/duration 반영까지 확인한다. |

Passing negative checks:
- Schema 밖 field 주입: `PATCH /api/scenes/s01 {outside_schema_p4:true}`는 `400 unsupported scene fields`.
- `sceneId` path 문자: encoded path segment `s01%2F..`는 `403 path traversal is not allowed`; render body `sceneId:"s01/../../scene_specs.json"`은 `404 render manifest has no scene`.
- Panel XSS: narration에 `<script>window.__P4_XSS=true</script>` 저장 후 실브라우저 form render에서 `window.__P4_XSS === false`, textarea value는 literal script string.
